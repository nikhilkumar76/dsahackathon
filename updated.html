üéØ Introduction to Queue

What is a Queue?

Queue is a linear data structure that follows the FIFO (First-In-First-Out) principle.

The first element added to the queue will be the first one to be removed.

‚úì Key Principle: First person to join the line is first to get served!

Real-World Analogies:

üé´ Ticket Counter

People join line ‚Üí First person served ‚Üí Exits line

üñ®Ô∏è Printer Jobs

Documents sent ‚Üí Printed in order ‚Üí Output in order

üìû Call Center

Calls received ‚Üí Answered in order ‚Üí Customer leaves

üè¶ Bank Counter

Customers queue ‚Üí First served ‚Üí Exit

Queue vs Stack:

Feature Queue (FIFO) Stack (LIFO)
Order First In, First Out Last In, First Out
Add Rear (Enqueue) Top (Push)
Remove Front (Dequeue) Top (Pop)
Real World Line, Printer, Browser History Plate stack, Browser Back Button

üì¶ Simple Queue (Linear Queue)

Definition:

A simple queue is the most basic type where elements are added at the rear and removed from the front.

Characteristics:

  • Add: Only at rear (end)
  • Remove: Only from front (beginning)
  • FIFO: First element added is first removed
  • Issue: Space wastage after dequeue

Problem with Simple Queue:

Space Wastage Example:

Initial: [1, 2, 3, 4, 5] (Capacity: 5)

After 2 dequeues: [_, _, 3, 4, 5] (2 empty spaces wasted!)

Can't add new elements at index 0,1 because front pointer moved!

Implementation (Pseudocode):

struct Queue { array[MAX_SIZE] front = 0 rear = -1 } Enqueue(x): if rear < MAX_SIZE - 1: rear++ array[rear] = x else: print("Queue Full") Dequeue(): if front > rear: print("Queue Empty") else: value = array[front] front++ return value Peek(): if front > rear: return "Empty" return array[front]

When to Use:

  • ‚úì Learning/Understanding queues
  • ‚úì When max size is small
  • ‚ùå Not ideal for production code
  • ‚ùå Inefficient for large datasets
üí° Solution: Use Circular Queue to solve space wastage problem!

‚≠ï Circular Queue

Definition:

A circular queue is an improved version where the rear connects back to the front, creating a circular structure. This eliminates space wastage.

Key Advantages:

‚úì No Space Wastage

Wraps around when reaching end

‚úì Better Memory Usage

Reuses empty spaces efficiently

‚úì Fixed Size

Works well with fixed-size arrays

‚úì O(1) All Operations

Enqueue, Dequeue, Peek all O(1)

How It Works:

Formula for circular movement:

next_position = (current + 1) % capacity

This wraps the index around when it reaches the end!

Circular Queue Implementation (Pseudocode):

struct CircularQueue { array[MAX_SIZE] front = -1 rear = -1 capacity = MAX_SIZE } Enqueue(x): if (rear + 1) % capacity == front: print("Queue Full") // No space else: if front == -1: front = 0 rear = (rear + 1) % capacity array[rear] = x Dequeue(): if front == -1: print("Queue Empty") else: value = array[front] if front == rear: front = rear = -1 else: front = (front + 1) % capacity return value Peek(): if front == -1: return "Empty" return array[front]

Conditions in Circular Queue:

Condition Formula Meaning
Empty front == -1 No elements
Full (rear + 1) % capacity == front One empty space reserved
Single Element front == rear Only one element
Next Front (front + 1) % capacity Move front circularly
Next Rear (rear + 1) % capacity Move rear circularly

When to Use Circular Queue:

  • ‚úì Fixed-size queue needed
  • ‚úì Memory efficiency critical
  • ‚úì CPU scheduling
  • ‚úì Printer buffer
  • ‚úì Traffic light management

üéØ Priority Queue

Definition:

A priority queue is a special type of queue where elements are served based on their priority rather than arrival order.

Higher priority elements are dequeued before lower priority ones.

Key Differences from Normal Queue:

Feature Normal Queue Priority Queue
Dequeue Order Arrival order (FIFO) By priority
Enqueue O(1) O(log n)
Dequeue O(1) O(log n)
Use Case Printer jobs Hospital emergency

üîÑ Queue Patterns & Algorithms

1Ô∏è‚É£ BFS (Breadth-First Search) Pattern

When to Use:

  • Finding shortest path in unweighted graph
  • Level-order tree traversal
  • Connected components
  • Multi-source shortest path

Pattern:

queue = new Queue() queue.enqueue(start) visited[start] = true while queue not empty: node = queue.dequeue() process(node) for each neighbor of node: if not visited[neighbor]: visited[neighbor] = true queue.enqueue(neighbor)

Complexity: O(V + E)

2Ô∏è‚É£ Sliding Window Pattern

When to Use:

  • Max/Min in sliding window
  • Fixed size window traversal
  • Moving average calculations

Pattern:

deque = new Deque() result = [] for i in 0 to n-1: // Remove elements outside window while deque not empty and deque.front < i - k + 1: deque.popFront() // Remove smaller elements while deque not empty and arr[deque.back] < arr[i]: deque.popBack() deque.pushBack(i) if i >= k - 1: result.push(arr[deque.front])

Complexity: O(n)

3Ô∏è‚É£ Multi-Source BFS Pattern

When to Use:

  • Find distance from multiple sources
  • Rotting oranges problem
  • Knight's shortest path
  • Prison cells problem

Pattern:

queue = new Queue() // Add all sources for each source in sources: queue.enqueue(source) visited[source] = true // BFS from all sources simultaneously while queue not empty: node = queue.dequeue() distance[node] = current_distance for each neighbor: if not visited[neighbor]: visited[neighbor] = true queue.enqueue(neighbor) current_distance++

Complexity: O(rows √ó cols)

4Ô∏è‚É£ LRU Cache Pattern (Deque + HashMap)

When to Use:

  • LRU Cache implementation
  • Recently used tracking
  • Frequency-based eviction

Pattern:

class LRUCache { cache = new HashMap() deque = new Deque() capacity = max_capacity get(key): if key not in cache: return -1 // Move to end (most recent) deque.remove(key) deque.pushBack(key) return cache[key] put(key, value): if key in cache: deque.remove(key) else if size == capacity: // Remove least recently used lru_key = deque.popFront() delete cache[lru_key] cache[key] = value deque.pushBack(key) }

Complexity: O(1) for all operations

5Ô∏è‚É£ Producer-Consumer Pattern

When to Use:

  • Thread-safe queue processing
  • Task distribution
  • Buffering data

Pattern:

class ProducerConsumer { queue = new Queue(capacity) produce(item): while queue.isFull(): wait() queue.enqueue(item) notify() consume(): while queue.isEmpty(): wait() item = queue.dequeue() notify() return item }

Used in: Thread pools, message brokers

6Ô∏è‚É£ Topological Sort Pattern

When to Use:

  • Dependency resolution
  • Course prerequisites
  • Build systems
  • Task scheduling

Pattern (Kahn's Algorithm):

inDegree = calculate_indegrees() queue = new Queue() // Add nodes with 0 in-degree for each node: if inDegree[node] == 0: queue.enqueue(node) order = [] while queue not empty: node = queue.dequeue() order.push(node) for each neighbor of node: inDegree[neighbor]-- if inDegree[neighbor] == 0: queue.enqueue(neighbor)

Complexity: O(V + E)
